# Contributing

This document provides guidelines for contributing to this project.

## Development Workflow

1.  **Create a new branch:** All changes should be made in a new branch. The branch name should be descriptive of the changes being made.
2.  **Make changes:** Make the necessary changes to the code.
3.  **Commit changes:** Commit the changes with a descriptive commit message.
4.  **Push changes:** Push the changes to the remote repository.
5.  **Create a pull request:** Create a pull request to merge the changes into the main branch.
